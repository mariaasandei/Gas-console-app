# Gas-console-app
# 🔥 GasCo Support System (Console App)

A robust C# Console Application designed to manage and streamline support tickets for a gas utility company. This project demonstrates **Object-Oriented Programming (OOP)**, role-based access, and basic data management.

---

## 🚀 Project Overview
This application serves as a communication bridge between customers (End Users) and the support team. It features a dual-interface system to handle gas-related inquiries and emergencies efficiently.

### 👥 User Roles & Permissions

#### 1. End User (Customer / Person needing support)
* **Create Tickets:** Report issues like gas leaks, low pressure, or billing errors.
* **Track Status:** View the progress of submitted tickets using a Unique Ticket ID.
* **Urgency Levels:** Flag tickets as *Low*, *Medium*, or *Emergency*.

#### 2. Support Staff (Person offering support)
* **Dashboard:** View a comprehensive list of all active and pending tickets.
* **Manage Lifecycle:** Update ticket statuses (e.g., *Open* -> *In Progress* -> *Resolved*).
* **Notes:** Add internal comments or instructions for technicians.

## 🛠️ Technical Specifications
* **Language:** C#
* **Framework:** .NET 8.0 (Console Application)
* **Design Patterns:** Separation of Concerns (Logic vs. UI), List-based data persistence.
* **Key Features:** * Auto-generated Ticket IDs.
    * Input validation for safety-critical reports.
    * Simple role-switching menu.
